## In five lines

Bring in the crate, feed three EIP-197 byte buffers, get a `bool`.

```rust
use zkmcu_verifier::verify_bytes;

let ok = verify_bytes(&vk_bytes, &proof_bytes, &public_bytes)?;
assert!(ok, "proof failed to verify");
```

That's it for the one-shot API. See [Getting started](/getting-started/) for the reusable parser variant and the embedded-target integration.

  </Card>

<Aside type="note" title="Why this matters">
  Every current ZK verifier ships for phones or servers. Hardware wallets, offline credential validators, IoT attestation tags — none of them run SNARK verification on-device today. zkmcu closes that gap on any 128 KB SRAM-class MCU.
</Aside>

## What it's for

<CardGrid>
  <Card title="Hardware wallets" icon="laptop">
    Local ZK verification without trusting the USB host. Fits the 128 KB SRAM class: Ledger ST33, Infineon SLE78, nRF52832, STM32F405.
  </Card>
  <Card title="Offline credentials" icon="seti:lock">
    Transit turnstiles, festivals, borders. No server round-trip needed to verify a privacy-preserving credential.
  </Card>
  <Card title="IoT attestation" icon="linux">
    Small devices verifying each other's provenance and claims directly, without a cloud relay.
  </Card>
</CardGrid>
